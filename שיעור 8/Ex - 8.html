Exercise #8

Multi-Dimensional Arrays


Exercise targets

     Work with Multi-Dimensional Arrays

What you should do

You are expected to create a calendar application(including the usual menu of course):

  • Define a matrix of strings with 12 lines (one line for each month) and enough columns to present all possible days of each month.

  • Create a menu with the following options:
    • Add event to calander
         1. Ask the user in which month and day he would like to add an event.
         2. Verify that the entered date actually exists, if it doesn't, return to the main menu.
         3. Ask the user to enter the details of the event.
          ** Each date can hold only one event. If the user tries to add an event to a date which already contains an event,
            ask him if he wishes to replace the old event with the new one.
    • Show all events in year
        Print all events in the year chronologically by months.
    • Show all events in a certain month (ordered chronologically).
** Each menu option will have a different function
** The function for option 2 will probably use the function of option 3
** To find out if there is no event in a certain date, one could check if the string is not null

Do not forget to write some documentations ( use // or /**/ )

Indentation is crucial (use the right tabing)

What you should not do

Whenever the user is prompt to insert a value, you can take from granted that the value inserted is valid. For example if the user is suppose to insert an integer and instead he inserts a string it is not up to you to handle the error (or at least not for this exercise).

Special guidelines

Please stick to the same indentation conventions we use in our examples and which the Visual Studio .NET help create. Use a clear and understandable names for your variables, for example the best name for the user's name will be (surprisingly) userName.
